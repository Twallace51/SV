# Reproduction Plan — Sendas de Vida Menu

This project can be reproduced by rebuilding the app in the same order the codebase evolved:

1. **Set up the runtime**
    - Create a Python environment with PySide6, pytest, and sqlite3-compatible tooling.
    - Confirm the app launches with `QT_QPA_PLATFORM=offscreen` for tests and a normal Qt session for manual use.

2. **Recreate the shared application shell**
    - Implement the package-level database helpers and active DB path management.
    - Build the login dialog, versioned main window, inactivity timer, backup action, and session/theme startup flow.
    - Keep the app running as a single instance and preserve the re-login-on-close behavior.

3. **Recreate the alumnos domain first**
    - Implement new/edit/search dialogs for alumnos.
    - Add grade lookup, live parent lookup, pension normalization, enrolled-only filtering, and numeric table sorting.
    - Restore the alumnos reports: por grados, becados, RUDE, carnet, cumpleaños, and parientes.
    - Add export support for reports where it exists in the current app.

4. **Recreate parientes support**
    - Implement new/edit/search dialogs for adultos/parientes.
    - Preserve current-record state so the UI can reflect the last saved or selected adult.
    - Add the missing report surface for this domain.

5. **Recreate cuentas support**
    - Implement new/edit/search dialogs for cuentas.
    - Preserve debito/credito validation, aclaración presets, and month-based pension insertion behavior.
    - Rebuild cuentas reporting, starting with totals and per-alumno balances.
    - Keep the search table behavior aligned with the UI expectation that alumno.id is visible first.

6. **Restore validation and integrity rules**
    - Enforce that `id_padre`/`id_madre` point to existing rows in `adultos`.
    - Enforce uniqueness for alumno RUDE and Carnet values during insert and update.
    - Keep edit flows permissive for the current alumno’s own values.

7. **Recreate tests alongside each feature**
    - Add dialog tests for create/edit/search behavior as each module is restored.
    - Add report tests for numeric totals, filtering, and export outputs.
    - Run targeted pytest slices after every feature area, then run the full suite.

8. **Finish with packaging and verification**
    - Confirm the main window menu structure, all actions, and the backup path.
    - Validate the app in both normal and headless modes.
    - Capture any missing parity items as follow-up work rather than blocking the baseline reproduction.

## Project Plan — Sendas de Vida Menu

### What the git history shows was built

**Foundation**
- Qt login dialog with trainee mode, password toggle, and secret admin shortcut
- Main window with title, version, inactivity timer, session theme, single-instance lock
- Logging, pip/pytest startup checks, backup action, About dialog

**Alumnos domain** *(most developed)*
- New / Edit (with grade lookup, parent ID + live lookup, integer pension) / Search dialogs
- Enrolled-only toggle, numeric sorting, `.title()` normalization on save
- Reports: Por grados, Becados, RUDE, Carnet, Cumpleaños, Parientes (all with print/CSV/Excel/Markdown export and date-stamped titles)

**Parientes domain**
- New / Edit / Search dialogs with current-record global state

**Cuentas domain**
- New / Edit / Search dialogs with debito/credito fields, aclaración selector, auto pension insert on month 1
- Reports: Total (global sum), Alumnos (per-alumno balance, non-zero only)

**Infrastructure**
- Trainee temp-DB session, re-login on close, role-based delete (admin only), database backup
